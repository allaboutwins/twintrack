## Prerequisites

Before starting Codemagic setup, you need:

1. **Apple Developer account** — enroll at developer.apple.com ($99/year)
2. **App Store Connect app record** — create the app at appstoreconnect.apple.com
   - Bundle ID: `com.allaboutwins.twintrack`
   - Name: TwinTrack
3. **GitHub account** — to push this repo and connect to Codemagic

---

## Step 1 — Push the Project to GitHub

Replit has a built-in git repo. You need to push it to GitHub so Codemagic can
access it.

In your Replit project:

1. Open the **Version Control** panel (git icon in left sidebar)
2. Click **Connect to GitHub**
3. Authorise Replit to access your GitHub account
4. Create a new private repo: `twintrack`
5. Push — Replit will push all commits to GitHub

Alternatively, from your terminal on any computer:

```bash
git remote add origin https://github.com/YOUR_USERNAME/twintrack.git
git push -u origin main
```

---

## Step 2 — Create App Store Connect API Key

This key lets Codemagic manage certificates, provisioning profiles, and upload
builds automatically.

1. Go to **App Store Connect → Users and Access → Integrations → App Store Connect API**
2. Click **+** to create a new key
3. Name: `Codemagic`
4. Access: **App Manager**
5. Click **Generate**
6. **Download the `.p8` file immediately** — Apple only lets you download it once
7. Note the **Key ID** and **Issuer ID** shown on that page

---

## Step 3 — Create a Codemagic Account and Connect GitHub

1. Go to **codemagic.io** → Sign up (free)
2. Connect your GitHub account when prompted
3. Click **Add application** → Select the `twintrack` repo
4. When asked for a workflow, select **Other** (you already have `codemagic.yaml`)

---

## Step 4 — Add the App Store Connect API Key to Codemagic

1. In Codemagic → **Teams → Personal Account → Integrations**
2. Under **App Store Connect**, click **Connect**
3. Enter:
   - **Key ID** — from Step 2
   - **Issuer ID** — from Step 2
   - **Private key** — paste the full contents of the `.p8` file
4. Name this integration: **`AppStore_API_Key`**
   (this must match `integrations.app_store_connect` in `codemagic.yaml`)

---

## Step 5 — Configure Automatic Code Signing

Codemagic handles certificates and provisioning profiles automatically using
your App Store Connect API key. No manual certificate export needed.

In App Store Connect, ensure:
- A **Distribution certificate** exists (Codemagic creates one if needed)
- An **App Store provisioning profile** for `com.allaboutwins.twintrack` exists

Codemagic's `xcode-project use-profiles` command fetches and installs these
automatically during every build.

> **Note:** If no distribution certificate exists yet, go to
> developer.apple.com → Certificates → create an **Apple Distribution**
> certificate. Codemagic can also create this for you if you grant it
> the right App Store Connect permissions.

---

## Step 6 — Set Up Environment Variables in Codemagic

In Codemagic → your app → **Environment variables**, add:

| Variable | Value | Secret? |
|---|---|---|
| `VITE_CLERK_PUBLISHABLE_KEY` | Your Clerk publishable key (starts with `pk_live_`) | Yes |
| `VITE_REVENUECAT_IOS_API_KEY` | RevenueCat iOS public key (starts with `appl_`) | Yes |
| `VITE_PREMIUM_ENABLED` | `true` (or `false` to disable paywall for testing) | No |

> ⚠️ **Do NOT add `VITE_CLERK_PROXY_URL`** to Codemagic. This variable points
> to your Replit dev server and is only used in the Replit preview. In the native
> iOS app, Clerk must connect directly to its own CDN — not through a Replit proxy.
> If this variable is set, authentication will fail on real devices.

> All other secrets (database credentials, session secret, etc.) live on your
> API server and are never baked into the iOS binary.

### Where to find these values

**Clerk publishable key:**
- Clerk Dashboard → your app → API Keys → Publishable key
- Use the `pk_live_...` key (not `pk_test_...`) for production builds

**RevenueCat iOS key:**
- RevenueCat Dashboard → your project → API Keys → Public app-specific key → iOS
- Starts with `appl_`

---

## Step 7 — Configure TestFlight Group

1. In App Store Connect → your app → **TestFlight**
2. Create an internal testing group named **`Internal Testers`**
   (must match `beta_groups` in `codemagic.yaml`)
3. Add yourself as a tester

---

## Step 8 — Trigger the First Build

1. In Codemagic → your app → click **Start new build**
2. Select branch: `main`
3. Select workflow: **TwinTrack iOS — App Store**
4. Click **Start build**

The build runs fully in the cloud on an Apple M2 Mac. It will:
- Install pnpm and all dependencies
- Build the React/Vite web app
- Run `npx cap add ios` (generates the Xcode project)
- Inject `ITSAppUsesNonExemptEncryption = false` into Info.plist automatically
- Run `npx cap sync ios` (installs CocoaPods including RevenueCat SDK)
- Sign the app using your App Store Connect API key
- Archive and upload to TestFlight

**Typical build time: 20–30 minutes**

---

## Step 9 — Verify on TestFlight

After the build succeeds:
1. Open **App Store Connect → TestFlight**
2. Your build will appear (may take 5–15 min for Apple to process)
3. Install on your iPhone via TestFlight app
4. Test all features end-to-end

---

## Step 10 — Submit to App Store

Once TestFlight testing is complete:
1. App Store Connect → your app → **App Store** tab
2. Select the TestFlight build
3. Fill in release notes
4. Submit for Review

## Build Commands Reference

What Codemagic runs (in order):

```bash
# 1. Install pnpm
npm install -g pnpm@latest

# 2. Install all workspace dependencies
pnpm install --frozen-lockfile

# 3. Build the React/Vite web app
pnpm --filter @workspace/twintrack run build
# Outputs to: artifacts/twintrack/dist/public

# 4. Generate iOS Xcode project (first build only)
cd artifacts/twintrack
npx cap add ios

# 5. Inject encryption compliance key
/usr/libexec/PlistBuddy -c "Add :ITSAppUsesNonExemptEncryption bool false" ios/App/App/Info.plist

# 6. Install CocoaPods + sync web assets into Xcode project
npx cap sync ios

# 7. Sign + archive + export IPA (Codemagic CLI)
xcode-project build-ipa --workspace ios/App/App.xcworkspace --scheme App
```

## RevenueCat Considerations

The `@revenuecat/purchases-capacitor` package is already installed. When
`npx cap sync ios` runs, it installs the RevenueCat iOS SDK via CocoaPods
automatically — no manual Podfile editing needed.

In your app code (wherever you initialise RevenueCat), use the iOS-specific
public SDK key from your RevenueCat dashboard:

```typescript
import { Purchases } from "@revenuecat/purchases-capacitor";

await Purchases.configure({
  apiKey: "appl_XXXXXXXXXXXX",  // iOS key from RevenueCat dashboard
  appUserID: user.id,           // Clerk user ID
});
```

## Clerk Auth Considerations

Clerk runs inside Capacitor's WKWebView using the standard web SDK. The current
Capacitor config has:

```typescript
limitsNavigationsToAppBoundDomains: true,
```

This restricts which domains the WKWebView can navigate to. If Clerk's OAuth
flows (Google Sign-In, Apple Sign-In) open external URLs and the app blocks
them, authentication will fail silently.

**Test on TestFlight first.** If sign-in works, no changes needed. If OAuth
redirects fail, either:
- Set `limitsNavigationsToAppBoundDomains: false` in `capacitor.config.ts`, or
- Add Clerk's domain to `WKAppBoundDomains` in Info.plist after first build

## Blockers Summary

| Blocker | Severity | Resolution |
|---|---|---|
| No Apple Developer account | 🔴 Hard stop | Enroll at developer.apple.com — $99/yr |
| Code not on GitHub | 🔴 Hard stop | Push via Replit Version Control panel |
| No App Store Connect app record | 🔴 Required before TestFlight | Create at appstoreconnect.apple.com |
| No App Store Connect API key | 🔴 Required for signing | Create in App Store Connect (Step 2) |
| RevenueCat iOS key not configured | 🟡 Runtime issue | Add key in app code before first TestFlight test |
| Clerk OAuth flows may be restricted | 🟡 Verify on TestFlight | Test after first build — may need config tweak |

## Future Updates (After First Build)

Every future update is two commands:

```bash
# In Replit — make your code changes, then:
git add -am "Your change description"
# Codemagic can auto-build on every push, or you trigger manually
```

In Codemagic you can enable **automatic builds on git push** — every push to
`main` triggers a new TestFlight build with an auto-incremented build number.
