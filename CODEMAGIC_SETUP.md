# PART A — Android / Google Play

**Fastest path to real users. No Mac, no Xcode, no provisioning profiles.**

---

## A1 — Create a Google Play Console account

1. Go to **play.google.com/console** → sign in with your Google account
2. Pay the **one-time $25 registration fee**
3. Complete the account details (name, address, phone)
4. You now have a Google Play developer account

---

## A2 — Create the TwinTrack app in Google Play Console

1. In Google Play Console → click **"Create app"**
2. Fill in:
   - **App name:** TwinTrack
   - **Default language:** English (United States)
   - **App or game:** App
   - **Free or paid:** Free
3. Tick both declarations at the bottom → click **Create app**

---

## A3 — Generate a keystore in Codemagic (no keytool needed)

The keystore is your Android signing certificate. Codemagic can generate one
for you — no Mac, no Android Studio, no terminal needed.

1. In Codemagic → click your **team name** (top-left) → **Code signing identities**
2. Click the **Android** tab
3. Click **"Generate keystore"**
4. Fill in:
   - **Keystore name:** `twintrack`
   - **Key alias:** `twintrack`
   - **Key password:** choose a strong password (save it — you'll need it)
   - **Keystore password:** choose a strong password (save it — you'll need it)
   - **Validity (years):** 25
5. Click **Generate** — Codemagic creates the keystore and shows you the
   base64-encoded value

6. Copy the **base64 keystore string** — you'll add it as `CM_KEYSTORE` below

> ⚠️ **Back up your keystore.** If you lose it, you can never update your app
> on Google Play. Download the `.jks` file from Codemagic and keep it somewhere safe.

---

## A4 — Create a Google Play service account (for automated uploads)

This lets Codemagic upload builds to Google Play automatically.

**In Google Cloud Console:**
1. Go to **console.cloud.google.com** → create or select a project
2. Enable the **Google Play Android Developer API**:
   - Search "Google Play Android Developer API" → click Enable
3. Go to **IAM & Admin → Service Accounts** → click **"Create service account"**
4. Name: `codemagic-play-upload` → click Create
5. Role: skip (click Continue)
6. Click **Done**
7. Click the service account you just created → **Keys** tab → **Add key →
   Create new key → JSON** → Download
8. The downloaded file is your service account JSON

**In Google Play Console:**
1. Go to **Setup → API access**
2. Click **"Link to Google Cloud project"** → select the project from above
3. Under Service accounts, click **"Grant access"** next to your service account
4. Role: **Release manager** → click Invite user → Send invitation

---

## A5 — Set up environment variables in Codemagic

In Codemagic → your TwinTrack app → **Environment variables**, add these as
**secret** variables:

| Variable | Where to get it | Secret? |
|---|---|---|
| `CM_KEYSTORE` | Base64 keystore string from Step A3 | Yes |
| `CM_KEYSTORE_PASSWORD` | Password you chose in Step A3 | Yes |
| `CM_KEY_ALIAS` | `twintrack` (alias from Step A3) | No |
| `CM_KEY_PASSWORD` | Key password from Step A3 | Yes |
| `GCLOUD_SERVICE_ACCOUNT_CREDENTIALS` | Contents of the JSON file from Step A4 (paste entire JSON as one value) | Yes |
| `VITE_CLERK_PUBLISHABLE_KEY` | Clerk Dashboard → API Keys → `pk_live_...` | Yes |
| `VITE_REVENUECAT_ANDROID_API_KEY` | RevenueCat Dashboard → your project → API Keys → Public app-specific key → Android (starts with `goog_`) | Yes |
| `VITE_PREMIUM_ENABLED` | `false` — keep off until subscriptions are ready to launch | No |

> ⚠️ **Do NOT add `VITE_CLERK_PROXY_URL`** — this Replit-specific variable
> must be absent for native builds. Clerk connects to its own CDN directly.

---

## A6 — Upload the first build manually (required one time only)

Google Play requires at least one manual upload before automated uploads work.

1. Trigger a build in Codemagic → **Start new build** → branch `main` →
   workflow **"TwinTrack Android — Google Play"** → Start
2. After ~20 minutes, download the `.aab` file from the Codemagic build artifacts
3. In Google Play Console → your app → **Internal testing** → **Create new release**
4. Click **Upload** → upload the `.aab` file
5. Click **Save → Review release → Start rollout**

After this first manual upload, every future Codemagic build uploads directly
to the internal testing track automatically.

---

## A7 — Add yourself as an internal tester

1. In Google Play Console → your app → **Internal testing**
2. Click **Testers** tab → **Create email list**
3. Add your own Google email address
4. Click **Save changes**
5. You'll get an opt-in link — open it on your Android device to install the app

---

## A8 — Publish to production (when ready)

1. Google Play Console → your app → **Production** → **Create new release**
2. Select the tested internal build → add release notes → Submit for review
3. Google reviews Android apps in a few hours to 1-2 days (much faster than Apple)

# PART B — iOS / App Store

**Requires solving code signing first. No Mac needed — Codemagic uses cloud Macs.**

---

## B1 — Prerequisites

- Apple Developer account ($99/year) — developer.apple.com
- App Store Connect app record with bundle ID `com.allaboutwins.twintrack`
- App Store Connect API key (App Manager role)

---

## B2 — Add the App Store Connect API Key to Codemagic

1. In Codemagic → **Teams → Integrations → App Store Connect** → Connect
2. Enter your Key ID, Issuer ID, and paste the `.p8` file contents
3. Name it exactly: **`AppStore_API_Key`**

---

## B3 — Create Distribution certificate and provisioning profile

Codemagic's automatic signing needs these to exist in your Apple Developer account.

**Option A — Let Codemagic create them:**
- In Codemagic → team → **Code signing identities** → iOS tab
- Click **Generate certificate** → select Apple Distribution
- Codemagic uses your App Store Connect API key to create the certificate
  on Apple's servers with no CSR or Mac required

**Option B — Create manually in Apple Developer portal:**
- Certificates → Apple Distribution → requires CSR (needs a Mac or Codemagic)
- Profiles → App Store Connect → select your bundle ID → Generate

---

## B4 — iOS environment variables in Codemagic

| Variable | Value | Secret? |
|---|---|---|
| `VITE_CLERK_PUBLISHABLE_KEY` | Clerk `pk_live_...` key | Yes |
| `VITE_REVENUECAT_IOS_API_KEY` | RevenueCat iOS `appl_...` key | Yes |
| `VITE_PREMIUM_ENABLED` | `false` | No |

> Do NOT add `VITE_CLERK_PROXY_URL` — leave it completely absent.

---

## B5 — Trigger the iOS build

1. Codemagic → Start new build → branch `main`
2. Workflow: **TwinTrack iOS — App Store**
3. Build takes ~25 minutes → uploads to TestFlight automatically

---

## B6 — Clerk auth note for iOS

The Capacitor config has `limitsNavigationsToAppBoundDomains: true`. Test
sign-in on your first TestFlight build — if Google OAuth fails, set this to
`false` in `artifacts/twintrack/capacitor.config.ts` and rebuild.

# Build Commands Reference

## Android build sequence
```bash
npm install -g pnpm@latest
pnpm install --frozen-lockfile
pnpm --filter @workspace/twintrack run build   # outputs to artifacts/twintrack/dist/public
cd artifacts/twintrack
npx cap add android                             # generates android/ project
npx cap sync android                            # copies web assets + installs plugins
cd android
./gradlew bundleRelease                         # produces .aab signed with keystore
```

## iOS build sequence
```bash
npm install -g pnpm@latest
pnpm install --frozen-lockfile
pnpm --filter @workspace/twintrack run build
cd artifacts/twintrack
npx cap add ios                                 # generates ios/ Xcode project
npx cap sync ios                                # copies web assets + installs CocoaPods
xcode-project build-ipa --workspace ios/App/App.xcworkspace --scheme App
```
